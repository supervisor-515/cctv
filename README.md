# CCTV 근무표 자동배정

부대 CCTV 근무표를 자동으로 편성하는 오프라인 웹앱입니다. 서버 없이 동작하며,
모든 데이터는 브라우저 localStorage에 저장됩니다.

## 실행

`index.html`을 브라우저로 열면 됩니다 (`engine.js`가 같은 폴더에 있어야 합니다).
별도 빌드·설치 과정이 없습니다.

## 파일 구조

| 파일 | 역할 |
|---|---|
| `index.html` | 화면(UI)·localStorage 저장/불러오기. DOM을 만지는 코드는 전부 여기에 있습니다. |
| `engine.js` | 배정 엔진·통계·검증. DOM에 의존하지 않아 Node에서 `require()`로 불러와 테스트할 수 있습니다. |
| `sync.js` | 서버 동기화(선택, Firebase). 관리자만 수정, 구성원은 실시간 열람 전용. |
| `firebase-config.sample.js` | 동기화 자동 설정 견본 — `firebase-config.js`로 복사해 사용 |
| `FIREBASE_SETUP.md` | 동기화 설정 가이드 (프로젝트 생성 → 계정 → 보안 규칙 → 연결) |
| `test/engine.test.js` | 엔진 회귀 테스트 (`node --test`) |

## 시간 구조

- 하루 = 06:30 ~ 익일 05:30, 1시간 24슬롯
- 주간: 06:30~16:30 (11슬롯)
- 저녁 고정: 17:30=다음날 밥교대(+순찰) · 18:30=다음날 상황병 · 19:30=다음날 당직 ·
  20:30=전날 상황병 · 21:30=전날 당직 · 14:30=평일 당일 상황병
- 야간 번초 4개 (각 4시간 간격 2슬롯): 1번초 22:30+02:30 · 2번초 23:30+03:30 ·
  3번초 00:30+04:30 · 4번초 01:30+05:30
- 17:00 추가 순찰: 2인 1조 중 '순찰만' 도는 1명 (0.5h 가산)

## 배정 규칙

**하드 제약 (절대 완화 안 됨)**
- 인접(연속) 슬롯 근무 금지 — 고정 슬롯 포함
- 전날 야간자의 아침(06:30/07:30/08:30) 근무 금지
- 같은 날 야간 번초 2개(이중 야간) 금지
- 열외자(주간/야간/전체)는 해당 영역에 배정 금지

**완화 사다리** — 인원이 부족하면 단계적으로 완화합니다.
1. tier 1: 제약 위반 없음 (같은 날 1인 1슬롯)
2. tier 2: 같은 날 중복 투입 허용
3. tier 3: 야간 이틀 연속 허용
4. tier 4: 부분 채움 — 그래도 못 채우면 하드 제약을 지키는 선에서 비워둠(미배정)

**공정성** — 누적 평균시간·그룹/슬롯/번초/야간/밥교대/순찰 배정률을 가중합한 점수가
낮은 사람부터 배정합니다(설정 탭에서 가중치 조정 가능). 백트래킹(MRV)으로 완전해를
찾은 뒤 1-opt(한 칸 교체)와 2-opt(두 칸 맞교환) 국소 개선을 돌립니다.
신병은 하루 2개 기본 배정으로 비교됩니다.

## 데이터

- 저장: localStorage 키 `cctv_roster_v2`
- 백업: [기본 설정] 탭에서 JSON 내보내기/불러오기 (구버전 백업 자동 마이그레이션)
- 사전등록(휴가·열외·당직/상황 예약)과 휴무일은 **생성 시점에만 반영**됩니다.
  이미 생성된 표에는 소급 적용되지 않으며, 대신 충돌이 달력·근무표 확인 탭에
  `사전등록 충돌` 경고로 표시됩니다.

## 서버 동기화 (선택)

기본은 localStorage 단독(오프라인)이지만, Firebase를 연결하면 **관리자(행보관)만
수정하고 구성원은 실시간 열람만** 하는 구조로 동작합니다.

- 관리자: 로그인 후 저장할 때마다 자동 업로드
- 구성원: 열람 계정 로그인 → 읽기 전용 화면(생성·관리 탭 숨김) + 실시간 갱신,
  근무표·달력·근무 카운트·공정성 지표는 전부 열람 가능
- 권한은 Firestore **보안 규칙**(읽기=로그인 사용자, 쓰기=관리자 이메일)으로 서버에서 강제
- 미설정·오프라인·로그아웃 상태에서는 기존과 완전히 동일하게 동작

설정 절차는 [FIREBASE_SETUP.md](FIREBASE_SETUP.md) 참고.

무작위 부대 구성에서도 깨지지 않는지를 중심으로 검증합니다. UI는 테스트하지 않습니다.

## 개발 메모

- `engine.js`와 `index.html`의 인라인 스크립트는 같은 전역 스코프를 공유합니다
  (`let DB`는 engine에서 선언, UI의 load/import가 재할당).
- 통계(`buildStats`)와 검증(`validateScheduleCached`)은 캐시되며, 저장(`save()`)
  시점에 무효화됩니다. DB를 직접 수정한 뒤에는 반드시 `save()` 또는
  `invalidateStats()`를 호출해야 합니다.
