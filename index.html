● = 근무표 생성됨 · ⚠ = 점검 필요(제약 위반·미배정) · 빨간 날짜 = 휴무일. 날짜를 누르면 그날 근무표 전체와 일정이 아래에 표시됩니다.

휴무일 지정

부대 휴무·전투휴무 등을 미리 지정하면 달력에 표시되고, [근무표 생성]에서 그 날짜를 열 때 ‘직장 휴무일’·‘다음날 휴무일’이 자동으로 체크됩니다. 대한민국 공휴일(2025~2027)은 자동 반영됩니다. 이미 생성된 표에는 소급 적용되지 않습니다.

일정 사전 등록

휴가·열외·당직·상황병을 미리 등록해두면 [근무표 생성]에서 그 날짜를 열 때 자동으로 반영됩니다. 휴가는 마지막날(복귀일)만 주간 열외로 적용되어 그날 야간부터 투입 가능합니다. 이미 생성된 표에는 소급 적용되지 않습니다.

근무표 생성

날짜·역할·열외를 지정하면 고정 슬롯을 먼저 박고, 밥교대·주간·야간을 백트래킹으로 자동 배정합니다. 전날 표가 있으면 전날/당일 당직·상황병·밥교대를 자동 이전합니다.

열외자 지정

이름을 탭하면 켜지고(주황), 다시 탭하면 꺼집니다. ‘모두 열외’는 해당일 분모(겪은 근무표 수)에서도 제외됩니다.

기간 일괄 생성

시작일부터 종료일까지 하루씩 차례로 자동 생성합니다. 휴무일·사전등록(휴가/열외/당직·상황 예약)이 자동 반영되고, 전날 표의 당직·상황병·밥교대가 다음날로 이어집니다. 한 번에 최대 31일.

근무 카운트 · 배정률

평균은 ‘달력 일수’가 아니라 각자 카운트 기준 이후 실제 생성된 근무표 수를 분모로 합니다. 비활성·전체열외 날은 분모에서 제외됩니다. 월별 보기는 해당 월에 생성된 근무표만 집계합니다(기존 누적시간 제외) — 신병이 상황병/당직으로 전환된 뒤 월 단위 현황을 볼 때 사용하세요.

CV(변동계수)·지니계수가 낮을수록 분포가 균등합니다.

배정률 = (그 그룹/슬롯에서 배정된 횟수) ÷ (그가 겪은 동일 그룹 근무표 수).

공정성 지표

활성·비신병 근무자 기준(신병은 의도적으로 많이 투입되므로 공정성 분산 계산에서 제외 — 카운트 표에는 그대로 표시됩니다). 각 항목의 인원·평균·표준편차·변동계수(CV)·최소~최대·지니계수. CV·지니가 낮을수록 균등합니다.

야간 배정률

형식: 횟수/분모, 배정률%. 분모는 각자 겪은 해당(야간/그룹) 근무표 수.

밥교대 배정률

형식: 횟수/분모, 배정률%. 분모는 밥교대 카운트 기준(전환 시에도 유지).

추가 순찰 배정률 (17:00·0.5h)

17:00 순찰은 2인 1조 — 다음날 밥교대자(17:30 근무자) 외에 ‘순찰만’ 도는 1명. 16:30 미투입·주간 가능자 중에서 평일과 주말·휴일을 따로 균등 카운트합니다. 형식: 횟수/분모, 배정률%.

시간대별 배정률

형식: 횟수/분모, 배정률%. ‘전체’는 분모=총 근무표 수, 그룹 선택 시 분모=해당 그룹 근무표 수·분자=그 그룹의 시간대 배정 횟수.

근무자 관리

구분(신병·당직·상황병·운항병·말년)·밥교대 가능 여부·기존 누적시간·활성/비활성 기간을 관리합니다. 신병→당직/상황병 전환 시 신병 근무 카운트는 초기화하고 밥교대 카운트는 유지합니다.
· 운항병(활성 1명): 월~목 09:30·13:30, 금 09:30·14:30 고정 + 금/토 야간 1회(균등). 상황병 근무도 서며(선택 시 겹치는 고정 자동 제외), 토·일 주간은 일반 근무자처럼 배정됩니다.
· 말년: 배정은 신병과 동일하게 취급하되(당직/상황병 미투입) 카운트 초기화는 하지 않습니다.

기본 설정

인접(연속) 슬롯 근무는 항상 금지됩니다(하드 제약 — 완화 불가). 전날 야간자의 아침 근무도 항상 금지됩니다.

공정성 점수 가중치 (낮을수록 우선 배정)

점수 = 누적평균시간·w + 당일이미받은시간·w + 그룹배정률·w + 슬롯배정률·w + (야간)야간배정률·w + 번초배정률·w − 신병보정. recruitBias가 음수면 신병을 약간 먼저 배정합니다.

백업 · 데이터

모든 상태는 이 브라우저의 localStorage에 저장됩니다. JSON으로 내보내 백업하고, 구버전 백업도 불러올 수 있습니다.

서버 동기화 (Firebase)

행보관(관리자)만 수정하고 구성원은 실시간 열람만 하도록 동기화합니다. 관리자가 저장하면 자동 업로드되고, 구성원 화면은 실시간으로 갱신됩니다. 미설정 시 기존처럼 이 브라우저에만 저장됩니다. 설정 절차: 저장소의 FIREBASE_SETUP.md 참고.

⚙ 연결 설정 (최초 1회 — FIREBASE_SETUP.md 참고)

실제 권한(읽기=로그인한 사용자만, 쓰기=관리자만)은 Firebase 콘솔의 Firestore 보안 규칙이 강제합니다 — FIREBASE_SETUP.md의 규칙을 반드시 적용하세요. 저장소에 firebase-config.js 파일을 두면 구성원은 이 설정 입력 없이 로그인만 하면 됩니다.
